Share Web App


Web Development

This is a web application that was created using Google App Engine, python, and Ajax. It updates latest post automatically with no need to refresh the page and only displays posts nearby to enhance efficient usage.

PROBLEM

Imagine you are writing a paper that is due in one hour on your laptop in the library, and suddenly, you realize your laptop has only 10% battery. You look around and find out that you are the only person in the room using a Macbook……

People tend to turn to their friends when they are in trouble even if their friends are not nearby. Although mobile phones can let them contact their friends immediately, their problems might not be able to wait until their friends show up. Therefore, in many cases, they have to turn to strangers, but it is not very effective and efficient for them to ask for help one by one.

It would be great if there is an application that is designed for people to get help from strangers easily.

SOLUTION

We created a web application using Python on Google App Engine. It is a single page web app that allows users to post their requests or have a little chat with people nearby.

  1. Users can connect their Facebook account so that they don’t have to enter their names every time they post.
  2. The posts show and vanish automatically upon its duration so that the page won’t be filled with expired posts.
  3. Only posts within certain miles are shown to keep the communication effective.

CHALLENGE

The posts are stored on Google Cloud Datastore. At first, there was a problem to show the latest post. After the user submitted the form and the page refreshed, the latest post didn’t show up and the user had to refresh the page manually again. I double checked the code many times and also consulted TA, and finally we figured that Datastore couldn’t finish updating the entry before the page refreshed. We didn’t want to spend too much class time on this issue so we set a short wait time before the page refreshed after the form was submitted. I wasn’t satisfied with that solution and believed there would be a better way.

In the early stage of the project, I was thinking about using Ajax to make the page update the latest post automatically without refreshing, but I decided to wait until we finished other features. The night after class, I was looking for a better solution for the Datastore problem. I found it worth trying to use Ajax to accomplish these two things the same time. So I spent a couple hours to work on Ajax and made it happen.
