Starlink Onboarding

A feature of Autel Robotics’ Starlink app that takes care of new user’s first time fly experience by providing interactive setup instructions and necessary fly tips.

June 2016 - August 2016, UX Design Intern

Starlink is Autel Robotics’ mobile app that enriches users’ drone ownership experiences. As a UX design intern, I partnered with another designer to develop high-fidelity mockups for the app’s onboarding feature that walks users through drone setup process and teaches them how to fly for the first time.

problem

Imagine you ordered your first drone a few days ago. You were so excited when the drone finally came and couldn’t wait for the first try, but you had no idea what to do. Then you found a Quick Start Guide and a user manual in the box. You opened the Quick Start Guide because it looked like what you need: a guide that is quick.

However...

  • you found the Quick Start Guide so unclear that you had to spend way longer time than expected to assemble the drone
  • you were frustrated by keeping failing the compass calibration
  • you were so confused by the icons and buttons on the app and regretted skipping the screen walkthrough.
When you finally got ready to fly, would you still be that excited?


What the user wants to fly for the first time:

  • An easy to read instruction that helps them get into the air quickly and complete some basic operations.
NOT:
  • A Quick Start Guide that is not quick
  • A user manual that provides too much information for the first fly
  • A couple of camera screens with overwhelming annotations that no one wants to read

goal

Before I joined, the idea of Onboarding was already generated, which aims to decrease total setup time and build a basic knowledge base for new users. My work focused on figuring out a smooth user flow for Onboarding and an efficient way to present information.

challenge

The setup process was defined by the hardware and software team together. Although I was aware of the complication of the setup process, there was no way to make changes to it but make it more comprehensive for users.

process

// Usability Testing //

Onboarding vs Quick Start Guide
I started with the usability testing of the Onboarding prototype version that was created by my partner before I joined to see if the Onboarding would be a promising way to achieve the goal and what problems the prototype had at that stage. We conducted an A/B testing on the Quick Start Guide and the Onboarding prototype because we wanted to compare the results and decide whether or not to continue working on Onboarding.

Results: 15min vs 30min, 80 points vs 63 points
After 16 rounds of tests (8 on Quick Start Guide and 8 on the Onboarding prototype), we were happy to see that on average the group who followed Onboarding spent only about 15 minutes to finish the setup process, while the group who followed Quick Start Guide needed over 30 minutes. What’s more, the SUS scores for Onboarding and Quick Start Guide were 80 and 63 respectively.


// Iteration 1 - Changes To Orientation & Flow //

With the happy results of the usability testing, the Onboarding got to be continued. Again, one of my responsibilities was to make sure the user flow is smooth. In order to achieve it, I suggested some changes that resulted in this iteration.

Change 1: turned the screen orientation from horizontal to vertical

Horizontal or vertical view has more effect on mobile phones than on tablets and it is more comfortable to read on a vertical screen when using a cell phone. Onboarding would be the first screen when the Starlink app is opened for the first time and mobile users are more likely to hold their phones vertically when they open the app.

I have also thought about keeping both vertical and horizontal views for Onboarding, but because of the time limit, I decided to leave the horizontal view design for the future.

Change 2: simplified the user flow and replaced static images with GIFs

The usability testing revealed some problems:

  • Static images didn’t provide enough information to build user’s confidence.
    People feel unsafe about the things they don’t know. For example, it needs some strength to pull out the battery. The first prototype used a static image to tell users how to pull the battery and users failed because they were scared of breaking the drone.

    Solution: Use GIFs to show the results of the behavior so that users know what they are expecting and be more confident.

  • Step flow wasn’t intuitive and users were confused by how the tabs were aligned with the circles.
    What most users did during the test was to open the first circle, go through all the tabs, close the tab window, and then open the second circle instead of going to the next screen.

    Solution: Simplified the overall navigation by removing circles and tabs and changing them to labels and details.


// Iteration 2 - Keep It Short & Improve Screen Walkthrough //

After showing the previous version to the manager, we were asked to add safety guidelines to the Onboarding. We agreed that it was needed, but adding content created a new challenge: the Onboarding was getting longer, which pushed our design away from the goal. We needed to keep the process short.

Make it look short
I read through all the steps to see if they could be combined. I also consulted the technicians in the company about important steps and knowledge for the setup and fly to make sure my changes were not misleading.

Make it feel short
Until then the Onboarding was still independent to the actual app so that the user had to finish the instructions before connecting and calibrating their drone, which means no matter how badly they wanted to try it on their own, they had to wait until they read everything. It might take 3 minutes in total, but it definitely feels more than that. Also, it’s more efficient and engaging to have users follow step by step.

Therefore, I suggested we break the onboarding content into two parts: the assemble part that only involves the drone and the connected part that the Starlink app is needed. In this way the Onboarding part not only feels shorter, but also is easier to follow.

Make annotations pleasant to read
As mentioned before, the original camera screen walkthrough was overwhelming. According to the usability testing, most users just skipped it because facing such a mess, they didn’t know where to start. So we introduced the camera screen by sections and it became easier to read.

// Settings Card Sorting //

The Starlink app allows users to make changes to the app system, the hardware, and camera. The same time the settings maximize the app’s customizability, they confuse the users. It’s not only because there is too much information under the setting button so that the users need a long time to find what they want, but also many settings are for advanced users only so that general users feel confused when reading those professional terms.
